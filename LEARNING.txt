┌─────────────────────────────────────────────────────────────────────────────┐
│  LEARNING #5 — TELEGRAM SESSION MANAGEMENT (WORKING)                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Telegram commands for Claude session lifecycle:                            │
│    claude   — start Claude session (tmux session named "brain")            │
│    exit     — signal Claude to run survive then exit                        │
│    survive  — signal Claude to run survive ritual only                      │
│                                                                             │
│  HOW IT WORKS:                                                              │
│  Bot: tmux new-session -d -s brain "bash -c 'cd /root && claude PROMPT'"   │
│  Wake prompt passed as positional argument to claude (not send-keys —      │
│  send-keys drops Enter for long strings with emoji)                         │
│  After 3s: send "1" Enter to auto-accept workspace trust dialog            │
│                                                                             │
│  Wake prompt tells Claude to:                                               │
│    1. git -C /root/learning pull                                            │
│    2. Read LEARNING.txt, CLAUDE_SELF.md, WHO_IS_ALEKS.md                   │
│    3. Read /root/duplex-bot-inbox.json → reply to unread messages          │
│    4. Send "Session restored. Ready." to Telegram                          │
│    5. Monitor journalctl -f -u duplex-bot for 📱 lines                     │
│                                                                             │
│  Inbox file: /root/duplex-bot-inbox.json  (NOT /var/log/ — outside         │
│  workspace, causes permission prompts)                                      │
│  Entry format: {type, text, time, read}                                    │
│    type=message → regular message from Aleks                               │
│    type=command text=exit → run survive then exit                          │
│    type=command text=survive → run survive only                             │
│                                                                             │
│  Session detection: tmux has-session -t brain                              │
│  Bot service: /root/duplex/scripts/bot.py                                  │
│                                                                             │
│  SIGNAL FLOW for exit/survive:                                              │
│    Aleks types "exit" → bot saves to inbox (type=command) → 📱 exit        │
│    logged to journalctl → Monitor fires → Claude reads inbox → acts        │
│                                                                             │
│  PERMISSION SETUP (root cannot use --dangerously-skip-permissions):        │
